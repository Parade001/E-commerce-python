# 📄 工单导出助手 (Ticket Export RPA)

本项目是一个自动化的工单抓取与 PDF 导出工具。它能够根据指定的时间范围，自动登录 OMS 系统，抓取符合条件的工单，并使用 Playwright 无头浏览器将工单详情导出为 PDF 文件。导出的文件会自动按照 **“责任人 > 责任原因”** 的目录结构进行归档。

**核心特性：**
* **全自动化**：自动登录、自动加密密码、精确分页抓取。
* **智能归档**：自动清理非法字符，按业务逻辑创建嵌套文件夹。
* **100% 绿色便携**：内置 Playwright 浏览器环境隔离机制，打包后无需客户机联网下载内核，完美绕过企业内网网络限制。

---

## 🛠️ 1. 开发环境配置 (开发者专属)

如果你需要修改代码并在本地测试，请按照以下步骤初始化环境：

### 1.1 安装依赖
确保你的电脑已安装 Python (推荐 3.9 - 3.12)。在终端中执行以下命令安装必要的 Python 库：
```bash
pip install requests playwright pyinstaller
```

### 1.2 初始化便携版浏览器内核
这是**极其重要**的一步。为了保证程序独立性，我们强制将浏览器内核安装在项目目录下的 `pw-browsers` 文件夹中，而不是系统的全局目录。
在项目根目录下执行：
```bash
python main.py install
```
*(注：该命令会下载约 100MB+ 的 Chromium 内核，取决于网速。成功后项目目录中会出现 `pw-browsers` 文件夹。)*

### 1.3 源码运行测试
确保同级目录下存在填写正确的 `config.ini` 文件，然后执行：
```bash
python main.py
```

## ⚙️ 2. 配置文件说明 (`config.ini`)

程序依赖同级目录下的 `config.ini` 文件来读取账号和查询时间。请在运行前确保该文件存在，内容格式如下：

```ini
[Credentials]
# 登录账号
account = 15819893256
# 登录密码（明文，程序会自动转换为 MD5）
password = your_password_here

[Query]
# 查询的起始时间 (格式: YYYY-MM-DD HH:mm:ss)
start_time = 2024-04-01 00:00:00
# 查询的结束时间 (格式: YYYY-MM-DD HH:mm:ss)
end_time = 2024-04-22 23:59:59
```

## 📦 3. 打包与分发流程 (生成发布版本)

当你完成代码修改，需要打包发给运营/业务同事使用时，请**严格按照**以下步骤操作：

### 第一步：执行打包命令
由于 Playwright 依赖底层的 Node 驱动，必须使用 `--collect-all` 参数将其完整打入 `.exe` 中。在终端执行：
```bash
pyinstaller -F main.py --collect-all playwright -n 工单导出助手
```
*编译完成后，你会在 `dist` 文件夹下看到 `工单导出助手.exe`。*

### 第二步：组装“绿色免安装包”
为了让同事**双击即用（不触发任何网络下载）**，你需要组装一个完整的运行环境。
在任意位置新建一个文件夹（例如命名为 `工单批量导出工具_V1`），并将以下**三个核心组件**放入其中：
1. `dist/工单导出助手.exe` (刚才打包出来的程序)
2. `config.ini` (配置文件模板)
3. `pw-browsers` (第一步中通过 `python main.py install` 生成的包含了离线内核的**文件夹**)

### 第三步：压缩分发
将整个 `工单批量导出工具_V1` 文件夹打成 `.zip` 压缩包，发送给你的同事。

## 🚀 4. 最终用户使用指南 (面向业务人员)

当非技术同事拿到你的 `.zip` 压缩包后，只需按以下步骤操作：

1. **解压**：将压缩包完整解压到电脑本地（建议解压到 D 盘或桌面，**不要**在压缩包内直接双击运行）。
2. **配置**：用记事本打开 `config.ini`，修改 `start_time` 和 `end_time` 为你需要导出的时间段。
3. **运行**：双击运行 `工单导出助手.exe`。
4. **获取结果**：喝杯咖啡 ☕。程序运行结束后，当前目录下会自动生成一个名为 `工单导出结果` 的文件夹，所有 PDF 都在里面。

## ⚠️ 5. 常见问题排查 (Troubleshooting)

| 报错现象 | 原因及解决办法 |
| :--- | :--- |
| `未找到配置文件: config.ini` | 用户没有将 `config.ini` 和 `.exe` 放在同一个文件夹内。 |
| `未检测到 Chromium 浏览器内核或初始化失败` | 打包分发时，忘记将 `pw-browsers` 文件夹与 `.exe` 放在一起。请确保它们同级存放。 |
| `登录接口连接失败` | 检查网络是否正常，或者 OMS 系统服务器 (`sw.cesaas.com:81`) 是否在维护。 |
| 黑窗口一闪而过 | 检查配置文件格式是否错误，建议在终端中手动运行 `.exe` 以查看具体报错信息。 |
